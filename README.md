# Claude Data Analyst Agent

Drop a CSV in Slack, @mention the bot, ask a question — get back a full report with charts in the thread.

## Why this exists

You could do this manually in Claude.ai: upload the file, prompt it, copy out the output. But then *you* have to do it every time, and so does everyone else on the team.

The bigger idea: **the knowledge lives in the system, not in individuals.** If five people need to analyze a file this week, all five just @mention the bot and get the same quality output. No one needs to know how to prompt Claude. No one needs to figure out the right format. No one needs to touch a tool they don't use.

You build the expertise once into the backend. Everyone benefits every time after.

---

This is built on Claude's [Managed Agents (beta) API](https://docs.anthropic.com/en/docs/managed-agents). The agent runs in a cloud environment, analyzes CSV data, and produces interactive HTML reports with Plotly charts. An optional Slack integration wires it into your team's workflow.

## Features

- **Cloud execution**: Agent runs Python in an Anthropic-managed cloud environment with pandas and plotly pre-installed
- **Interactive HTML reports**: Generates self-contained reports with 3+ embedded Plotly charts and actionable recommendations
- **File mounting**: CSV files are uploaded via the Files API and mounted into the session at runtime
- **Environment/agent reuse**: IDs are persisted in `.env` so reprovisioning is skipped on subsequent runs
- **Slack bot** (optional): Mention the bot in any channel to kick off an analysis; follow-up replies continue the same session

## Project Structure

```
data_analyst_agent.ipynb   # Step 1 — provision cloud env/agent, run one-shot analysis

sample_data/

```

## Prerequisites

- Python 3.10+
- An [Anthropic API key](https://console.anthropic.com/)
- (Optional) A Slack app with Bot Token and App-Level Token for the Slack bot

## Installation

```bash
pip install "anthropic>=0.91.0" python-dotenv slack_bolt requests markdown-to-mrkdwn pandas plotly
```

## Quick Start

### Step 1 — Run the data analyst notebook

Set your API key, then open and run `data_analyst_agent.ipynb` cell by cell:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
jupyter notebook data_analyst_agent.ipynb
```

The notebook will:
1. Create a named cloud environment with pandas + plotly installed
2. Create a versioned Claude agent (`claude-sonnet-4-6`) with a professional analyst system prompt
3. Upload `sample_data/fifa_wc_data.csv` to the Files API and mount it in the session
4. Stream agent events (tool calls, text output) to the terminal
5. Download the generated `report.html` to the project root
6. Save `ANALYST_ENV_ID`, `ANALYST_AGENT_ID`, and `ANALYST_AGENT_VERSION` to `.env`

Open `report.html` in a browser to view the interactive analysis.

### Step 2 (optional) — Run the Slack bot

Requires the IDs written by Step 1. On first run you will be prompted for your Slack tokens:

```bash
jupyter notebook slack_data_bot.ipynb
```

Then in Slack:
- **Start an analysis**: `@YourBot analyze the attached sales.csv`
- **Attach a CSV**: Upload a file in the same message — it will be mounted automatically
- **Follow up**: Reply in the same thread to continue the conversation in the same session

The bot streams progress updates and uploads the final report and any generated scripts back to the thread.

## Environment Variables

| Variable | Set by | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | You | Required before first run |
| `ANALYST_ENV_ID` | `data_analyst_agent.ipynb` | Reused cloud environment ID |
| `ANALYST_AGENT_ID` | `data_analyst_agent.ipynb` | Reused agent ID |
| `ANALYST_AGENT_VERSION` | `data_analyst_agent.ipynb` | Agent version number |
| `SLACK_BOT_TOKEN` | `slack_data_bot.ipynb` (prompt) | Slack bot OAuth token |
| `SLACK_APP_TOKEN` | `slack_data_bot.ipynb` (prompt) | Slack app-level token (Socket Mode) |

## Sample Data

`sample_data/fifa_wc_data.csv` contains 48 teams competing in the FIFA World Cup 2026 with 24 columns including squad market value, FIFA ranking, historical stage progression (round of 16, QF, SF, Finals), goals scored/received over the last 4 years, and more. Use it to test the agent end-to-end out of the box.

## Architecture Notes

- **Session-per-thread**: The Slack bot maps each `thread_ts` to a `session_id`, enabling multi-turn follow-ups without reprovisioning
- **Streaming relay**: Agent events (`agent.message`, `agent.tool_use`, `session.status_idle`) are consumed and translated to Slack `mrkdwn` format in real time
- **Tool configuration**: The agent has `agent_toolset_20260401` enabled with `web_search` and `web_fetch` disabled — it only has access to bash, file read/write, and Python execution
- **Agent system prompt**: The primary tuning point for analysis style, chart preferences, and report structure — edit `ANALYST_SYSTEM_PROMPT` in `data_analyst_agent.ipynb`
